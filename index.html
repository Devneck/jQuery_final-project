This is the Final Project for the JavaScript/jQuery class. This tab is the home and was used as the starting point for the Tabs you will see above. The tab widget is pretty neat and fairly simple to use. I think I actually spent more time playing with the different themes and trying to figure out what I wanted this page to look like than actually setting up the tabs. In the end, I ended up selecting one of the provided themes that I liked best.

This tab is used for the Carousel and Lightbox plug-ins. I had a lot of fun with these 2 plug-ins and can definitely see me looking for a reason to use each of these. A couple things I did notice about these 2 plug-ins though. On the Carousel, there is a defect with how the arrows progress when navigating to the right which will not jump from the first image to the last image for displaying the title and alt text. No worries though, as someone had provided the code on how to correct the issue in the comments section. With the Slimbox2 plug-in (used as the Lightbox), the html code required to make it work does not validate. This isn't much of an issue since it works nonetheless, but I would be curious if the author is exploring other options that would allow the page to validate.

The Date Picker widget ... this particular piece of the assignment was the bane of my existence. I had a very tough time getting this to work the way the requirements called for it. Particularly, manipulating the start and end dates on the return date based on the selection in the departure date. This tab focuses on Tooltips and Tabs, or more specifically adding Tabs. The tooltips were very easy to implement, basically just turn them on (initialize). Then it was simply a matter of styling to look nice. I feel like I struggle with the "look" part of a website because I am not really a graphic person but I generally feel pretty good about the code which has nothing to do with graphics. I don't know if this is what was expected in terms of styling the tooltips, I just changed a few things so that I could show that I understood how to do that. The adding of a new tab was fun for me. It wasn't until I finished hand coding the entire add process flow when I realized the requirements reference a code example we could use and modify. I think it was better for me to actually do it myself rather than change someone else's code. After looking at the example, I actually like my way better because I perform client side validation on the fields prior to adding the tab where the example would add the tab even when the fields are left blank. My method seems more realistic to me as far as end result. By the way, feel free to add a tab now with the button below.

This final tab (although there is probably another now that you added one from the previous tab) is using the jLayout plug-in. I wasn't really too impressed with this plug-in, I guess the various layouts I did not select might have made it more entertaining but it really just seems to me this could be handled quite easily with styles or replicated without the plug-in quite simply. I did have to add a height and the overflow to the entire page specifically for this tab. Without it, then the entire containing div shifted as a result of the scrollbar on the right. I'm not really fond of it being there and would probably explore different options if I had more time to work on this. The method I used just reminds me of frames a little too much...
